# Git 文件四种状态

## 一、状态一览

| 状态 | 场景 | 含义 |
|------|------|------|
| **未跟踪** (Untracked) | 新建了 `Student.java` | Git 还没登记这个文件，不关心它的变化 |
| **未修改** (Unmodified) | 代码已成功 commit | 当前文件和仓库最新版一致 |
| **已修改** (Modified) | 打开文件多写了两行代码并保存 | Git 发现文件被改过，但还没放进暂存区 |
| **暂存** (Staged) | 准备把修改正式打包提交 | 文件进了"待发货区"，像淘宝购物车还没付款 |

## 二、状态转换

| 操作 | 从 → 到 | 说明 |
|------|---------|------|
| `git add <file>` | 未跟踪 → 暂存 | 新文件直接进暂存区（跳过中间状态） |
| `git commit` | 暂存 → 未修改 | 暂存区的文件打包进仓库，变回干净状态 |
| `git rm <file>` | 未修改 → 未跟踪 | 删除受管文件，变回和 Git 无关 |
| 修改文件并保存 | 未修改 → 已修改 | 用编辑器改了文件内容 |
| `git add <file>` | 已修改 → 暂存 | 再次 add 放进购物车，等下次 commit |

## 三、避坑指南

改了代码直接 `git commit` → **什么都不会提交**。

因为修改过的文件必须先 `git add` 进入暂存状态，才能 `git commit` 入库。
